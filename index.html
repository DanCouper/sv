Coding and designing a blog

Wednesday, 28th May 2020

Let's say I want to write an article on {subject}. Scratch that – I want to start a blog, because ego. This think pops into my head several times a month, but never actually reifies.

Full dislosure: I am a programmer who came to programming from visual design, so my intentions are doubly fucked due to requiring both a perfect design and a perfect codebase.

So I'm going to do that again, and I'm going to walk through it. I'll start with a very basic HTML, then gradually add some styling, then abstract the end HTML out into something repeatable that I don't have to think about. I'm trying to stick to the principle of YAGNI; to avoid coding myself into a corner I will need to make a few optimisations based on what I expect to happen in the future, and at some point I will need to introduce some JS (I just don't really fancy figuring out the HTML/CSS structure I'd need to make the checkbox hack work to run all of the functionality on this page)

HTML Structure

I have a site header, then a main area in which there's one article (this one). That article has a header containing the title and some meta information. And that article is divided into sections. Each section has a header and a div tag containing the body content. TODO: toggle to turn document structure on and off (use CSS to display)

Initial styling

Firstly, the text runs across the full width of the page. This isn't too much of an issue if you're looking at this on mobile, but it's definitely not great if you're looking at it on a desktop. I want to limit the length of the lines of text to something a bit more readable, and I'd like the text to sit in a column down the middle of the page. Easy: 

body {
  margin: 0 auto;
  max-width: 50rem;
}

Secondly, I like serif text, but not the default Times New Roman (you won't see this if you're viewing in mobile) -- it's condensed and it all looks like a Microsoft Word document from the late 1990s. I'm going to go with sans-serif anyway for now, and I'm going to add what I guess you'd call the system UI defaults. These fonts are generally whatever is used for UI on any given system. They are designed primarily to make text on controls easy to read, but tend to work pretty well for body text. I want a very clean look, and they will work well for that. 

:root {
  box-sizing: border-box;
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen,
Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif;
}

I could stop here. This works: it isn't fantastic by any means, but it's clean and pretty readable. However, I'm not going to stop here because design.

Initial Typography

Typography is pretty important when you want to make something look readable and professional. Like programming, it engenders extremely fierce tribal loyalties, causes raging battles over minor points and is exceptionally boring to outsiders. I'll kinda skip over most of the theory and say it's basically how you space out letters on a page.

So I have, within an article, an overall title, then a series of sections, each with their own title. I want those section spaced out slightly: they seem to run onto each other too much. And for good measure, add a bit of padding to the body. 

*,
*::before,

*::after {
  box-sizing: inherit;
  padding: 0;
  margin: 0;
}

body {
  line-height: 1.5;
  margin: 0 auto;
  max-width: 50rem;
  padding: 4rem 1rem;
}

article > header,
section:not(:last-child) {
  padding-bottom: 2rem;
}

section > header > h1,
section > div > p {
  font-size: 1rem;
  padding-bottom: 1rem;
}

Adding classes

Applying styles to bare, nested elements is going to get annoying fast, so time to add some classes and keep everything nice and flat.

At the same time, I'll adjust the HTML a bit – instead of just centring the body, I'm going to centre content within the main components of the page (and remove the padding from the body). The reason for doing this is that I can then control the background and horizontal borders on a per-component basis. It means I need wrapper elements, but it's a lot more flexible. 

body {
  line-height: 1.5;
}

.site-header,
.article {
  margin: 0 auto;
  max-width: 50rem;
  padding: 0 1rem;
}

.site-header,
.article-header,
.article-section:not(:last-child) {
  padding-bottom: 2rem;
}

.article-section-title,
.article-section-body p {
  font-size: 1rem;
  padding-bottom: 1rem;
}

As an example, I want a very thin border beneath the blog title, just to split it off visually from the rest of the page. 

.site-header-wrapper {
  border-bottom: 1px solid black;
}

Ugh, horrible

There are a few issues here:

  1. The site header looks awful – the spacing is all wrong, and the black of the border, despite being the same colour as the text, is too strong.
  2. I want the first article section to be an overview, so it needs to move to the article header.
  3. Code blocks cause the page to scroll horizontally.
  4. The only text elements I've really thought about are the paragraphs and headers – not, for example, lists.

I'll fix these one by one. I'll start by adding some variables for colour and spacing. 

:root {
  --colour-bg-base: hsla(0, 0, 88, 1);
  --colour-fg-base: hsla(0, 0, 8, 1);
  --colour-fg-base-faded: rgba(0, 0, 8, 0.1);

  --spacing-small: 0.5rem;
  --spacing-base: 1rem;
  --spacing-large: 1.5rem;
  --spacing-xlarge: 2rem;

  --font-size-small: 0.75rem;
  --font-size-base: 1rem;
  --font-size-large: 1.25rem;
  --font-size-xlarge: 1.5rem;
  --font-size-xxlarge: 2rem;

  --font-family-base: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
  Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif;

  font-family: var(--font-family-base);
  color: var(--colour-fg-base);
  background-color: var(--colour-bg-base);
}

I then replace values in the stylesheet with the variables. I don't need to do this, but it makes it hella easier to adjust values.

Site Header

This needs a lot more work, but for now:

  • Add a branding element, with an SVG logo with fills using the colour variables.
  • Set the title to a defined size.
  • Give it a little room, and add a margin between it and the main body.
  • Set the display to flex and push logo and title to opposite ends.
  • Set the alpha channel of the border colour to 0.1 (10%).
.site-header-wrapper,
.site-body {
  border-bottom: 1px solid var(--colour-fg-base-faded);
  margin-bottom: var(--spacing-xlarge);
}

.site-header,
.article {
  margin: 0 auto;
  max-width: 50rem;
  padding: 0 var(--spacing-base);
}

.site-header {
  display: flex;
  align-items: center;
  justify-content: space-between;
}

.site-title {
  font-size: var(--font-size-xxlarge);
}

.site-branding {
  width: calc(1.5 * var(--spacing-xlarge));
}

.site-logo-bg {
  fill: var(--colour-fg-base);
}

.site-logo-fg {
  fill: var(--colour-bg-base);
}

Article Header

The article header will be used as the overview for the article. So it needs to stand out slightly. It goes in the header, and has the same basic styling as section text. To make it stand out slightly, I'll apply an oblique styling to the text. 

.article-overview {
  padding: var(--spacing-small) 0;
  font-style: oblique;
}

.article-overview > * {
  padding-bottom: var(--spacing-small);
}

.article-section-title {
  font-size: var(--font-size-large);
}

.article-section-title,
.article-section-body > * {
  padding-bottom: var(--spacing-base);
}

Note that I've selected all direct children to apply padding -- this takes care of spacing between all block-level section children.

Code blocks

This uses a port of lolight.js, loaded as a module & used as a progressive enhancement. I'm also loading in a light version of Fira Code to use as the font face -- light because if needs to sit slightly back from the text. The code blocks sit in wells on the page, and back up the text rather than being the sole focal point. I'm not happy with this at the minute, so this is basically a placeholder.
